<objective>
Improve off-route-at-start navigation UX for imported Strava (waypoint) routes. When a rider taps START NAVIGATION far from the route start (the normal at-home case), NavigationManager currently fires an immediate reroute through ~200 remaining waypoints, which can fail and degrade to a "Follow route" straight lead-in. This plan makes the waypoint-route path distinguish "haven't reached the route yet" from "deviated off a route I was on": it only reroutes AFTER the rider has joined the route, shows "Head to route → {dist}" while approaching, and caps mid-ride reroute waypoints for reliability.

PHONE-ONLY. Only NavigationManager.kt + NavigationRouteTest.kt change. No glasses/shared/protocol changes. The destination-only (non-waypoint) reroute path — `calculateRoute(lat, lng, destLat, destLng, ...)` — is UNCHANGED (it starts its route from the current position and is always on-route at start).

Purpose: Fix the flagged off-route-at-start UX (STATE.md "Open follow-up") without regressing NAVV-03, the WR-01 atomic-publish discipline, Pitfall 3 forward-only indices, or the D4 full=false birdview invariant.
Output: hasBeenOnRoute gating, "Head to route" approach branch, a pure/testable reroute-waypoint cap, and three new unit tests — all under one build+test gate.
</objective>
