## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS AlarmManager / broadcast plane → app | Watchdog broadcasts can be spoofed if the receiver is exported |
| Manifest permission grants → OS | New background-capability permissions expand the app's attack/abuse surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Elevation of Privilege | WatchdogReceiver | mitigate | android:exported="false" (only the OS alarm PendingIntent reaches it); action-string check; no-op unless rec_active |
| T-06-02 | Elevation of Privilege | Watchdog PendingIntent | mitigate | PendingIntent.FLAG_IMMUTABLE on the broadcast intent (repo convention, ASVS V4) |
| T-06-03 | Elevation of Privilege | ACCESS_BACKGROUND_LOCATION + exact-alarm declarations | mitigate | Background location is used ONLY by OS-initiated recovery paths (never continuous background collection beyond the user-started recording); exact alarms scheduled only while recording, chain self-terminates via rec_active guard |
| T-06-04 | Denial of Service | Exact-alarm revocation force-stop (Pitfall 2) | mitigate | canScheduleExactAlarms() gate + inexact fallback; USE_EXACT_ALARM preferred on 33+ (auto-granted, non-revocable); L3 orphan recovery absorbs a force-stop mid-recording |
| T-06-05 | Denial of Service | ColorOS kill despite correct code (Pitfall 4) | accept + detect | Cannot be prevented dev-side (dontkillmyapp: no known solution); layered detection/recovery (L1/L2/L3) + user-side checklist in plan 01-07's device verification |
