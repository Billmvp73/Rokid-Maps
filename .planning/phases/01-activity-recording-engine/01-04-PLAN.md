## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Phone → paired BT clients | sport_state JSON broadcast over unencrypted SPP |
| OS/lock screen → bystander | FGS notification content visible without unlock |
| OS lifecycle → service | Background restarts (START_STICKY, watchdog) cross the Android 14 FGS-type permission boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | sport_state over BT SPP | accept | Existing accepted protocol risk (paired link); sport_state adds metrics only — NO coordinates (StateMessage already carries them); disposition documented per phase security requirement |
| T-04-02 | Information Disclosure | Lock-screen notification text | mitigate | Notification shows only distance + elapsed (activity metrics), never location/coordinates; channel is existing IMPORTANCE_LOW |
| T-04-03 | Denial of Service | startForeground SecurityException crash-loop on background restart | mitigate | try/catch(SecurityException) → Log.e + stopSelf + START_NOT_STICKY (Pitfall 1); recovery via orphan checkpoint on next user launch |
| T-04-04 | Tampering / Integrity | Metrics divergence (saved ≠ broadcast) | mitigate | Single MetricsSnapshot per tick feeds BT + UI + notification + checkpoint (ARCHITECTURE anti-pattern 5 enforced structurally) |
| T-04-05 | Elevation of Privilege | Notification contentIntent PendingIntent | mitigate | FLAG_IMMUTABLE (existing buildNotification convention, ASVS V4) |
