<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| phone → strava.com (HTTPS) | GPX upload body carries precise location+time history — HTTPS mandatory |
| Bearer token → Authorization header | Token material from the Phase-3 encrypted store; must never be logged |
| Strava upload/poll response → UploadResponse | Untrusted external JSON parsed all-nullable; status/error strings drive control flow |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Information Disclosure | GPX (location history) in transit | mitigate | HTTPS-only via the OkHttp client (no plaintext fallback); the upload body is never logged; the DEBUG interceptor stays Level.BASIC (line only, no body) — mirrors Phase-3 discipline |
| T-05-06 | Information Disclosure | Bearer token / upload body via logging | mitigate | logRateLimits logs only header usage; interceptor DEBUG-only BASIC; no Log statement emits the token, GPX, or coordinates (grep gate) |
| T-05-07 | Tampering | Manual multipart boundary corrupting the body | mitigate | MultipartBody.Builder + setType(FORM) — OkHttp owns the boundary; NO manual Content-Type header (grep gate = 0) — prevents the data:empty failure |
| T-05-08 | Denial of Service | Malformed/oversized upload response | mitigate | Gson parse in try/catch (never rethrow); all-nullable UploadResponse tolerates missing fields; poll() is a single stateless call (the 2-min deadline lives in Wave 3's driver) |
| T-05-09 | Elevation of Privilege | 401 refresh loop at upload | mitigate | ensureFreshToken proactive PRIMARY; the reactive StravaAuthenticator's responseCount>=2 give-up (Phase 3) prevents a refresh-loop DoS; token endpoints stay on the plain client |
| T-05-SC | Tampering | package installs | mitigate | No packages added — OkHttp 4.12.0 + Gson 2.10.1 already audited (Phases 1–4); RESEARCH audit records zero additions |
</threat_model>
