<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| recorded track → GPX string | Local trusted data (own recording) serialized in-memory; never written to shared storage in this plan |
| Strava error string → parseDuplicateActivityId | Untrusted Strava-supplied text parsed into a bounded numeric id (this plan owns the parser; the network call is Wave 2) |
| session JSON file ↔ updateUploadState | The source-of-truth file is read-modified-written; a mid-write failure must not corrupt it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | updateUploadState in-place file corruption | mitigate | Reuse the tested `writeAtomic` (temp+fsync+rename); never FileWriter-overwrite the source of truth — a crash mid-write leaves the old valid file (UPL-03 structural) |
| T-05-02 | Tampering | Duplicate-error string injection via parseDuplicateActivityId | mitigate | Regex captures only `\d+` → `toLongOrNull`; a non-numeric or malicious string yields null (Failed), never an arbitrary side effect; the id is a Long, never echoed as markup |
| T-05-03 | Information Disclosure | GPX body (precise location+time) leaking to logs/disk | mitigate | GpxWriter returns a String held in memory only; this plan writes NO GPX file and logs NO GPX/coordinate content (only Log.w on parse failure with `e.message`) |
| T-05-04 | Denial of Service | Malformed GPX/oversized re-parse in isValidForUpload | mitigate | XmlPullParser resolves no external entities by default (XXE-safe, mirrors GpxParser); parse wrapped in try/catch → false; input is our own output, bounded by track size |
| T-05-SC | Tampering | npm/pip/cargo/gradle installs | mitigate | No new packages added — kxml2/org.json/JUnit4 already audited in Phases 1–4; RESEARCH Package Legitimacy Audit records zero additions. No install task in this plan. |
</threat_model>
